## Value

Switches are not selected by default. The `defaultSelected` prop can be used to set the default state (uncontrolled). Alternatively, the `isSelected` prop can be used to make the selected state controlled.

```tsx render hideImports
"use client";
import {Switch} from '@react-spectrum/s2';
import {useState} from 'react';

function Example() {
  let [selected, setSelected] = useState(false);

  return (
    <div style={{display: 'flex', flexDirection: 'column', gap: 12}}>
      <Switch defaultSelected>
        Low power mode (uncontrolled)
      </Switch>

      <Switch
        isSelected={selected}
        onChange={setSelected}>
        Low power mode (controlled)
      </Switch>
    </div>
  );
}
```

## Events

Switches accept a user defined `onChange` prop, triggered whenever the Switch is pressed. The example below uses `onChange` to update state and show the current switch status.

```tsx render hideImports
"use client";
import {Switch} from '@react-spectrum/s2';
import {useState} from 'react';

function Example() {
  let [selected, setSelected] = useState(false);

  return (
    <div style={{display: 'flex', flexDirection: 'column', gap: 12}}>
      <Switch onChange={setSelected}>
        Wi-Fi
      </Switch>
      <div>Wi-Fi is: {selected ? 'On' : 'Off'}</div>
    </div>
  );
}
```

## HTML forms

Switch supports the `name` and `value` props for integration with HTML forms.

```tsx render hideImports
"use client";
import {Switch} from '@react-spectrum/s2';

<form>
  <Switch name="power" value="low">Low power mode</Switch>
</form>
```
