### Toast Variants

Toasts support four different variants to communicate different types of messages:

```tsx render expanded
"use client";
import {Button, ButtonGroup, UNSTABLE_ToastQueue as ToastQueue} from '@react-spectrum/s2';

  <Button
    onPress={() => ToastQueue.neutral('Toast available')}
    variant="secondary">
    Show Neutral Toast
  </Button>
  <Button
    onPress={() => ToastQueue.positive('Toast is done!')}
    variant="primary">
    Show Positive Toast
  </Button>
  <Button
    onPress={() => ToastQueue.negative('Toast is burned!')}
    variant="negative">
    Show Negative Toast
  </Button>

  </Button>

```

### Accessibility

Toasts are automatically displayed in a landmark region labeled "Notifications". The label can be overridden using the `aria-label` prop of the `ToastContainer` element. Landmark regions can be navigated using the keyboard by pressing the F6 key to move forward, and Shift + F6 to move backward. This provides an easy way for keyboard users to jump to the toasts from anywhere in the app.

### Internationalization

To internationalize a Toast, all text content within the toast should be localized. This includes the `actionLabel` option, if provided. For languages that are read right-to-left (e.g. Hebrew and Arabic), the layout of Toast is automatically flipped.

## Events

Toasts can include an optional action by specifying the `actionLabel` and `onAction` options when queueing a toast. The `shouldCloseOnAction` option automatically closes the toast when an action is performed.

```tsx render expanded
"use client";
import {Button, UNSTABLE_ToastQueue as ToastQueue} from '@react-spectrum/s2';

<Button
  onPress={() => ToastQueue.info('An update is available', {
    actionLabel: 'Update',
    onAction: () => alert('Updating!'),
    shouldCloseOnAction: true
  })}
  variant="primary">
  Show actionable toast
</Button>
```

### Close Events

The `onClose` event is triggered when the toast is dismissed, either by user action or programmatically:

```tsx render expanded
"use client";
import {Button, UNSTABLE_ToastQueue as ToastQueue} from '@react-spectrum/s2';

<Button
  onPress={() => ToastQueue.positive('Settings saved!', {
    onClose: () => alert('Toast was closed')
  })}
  variant="primary">
  Show toast with close handler
</Button>
```

## Auto-dismiss

Toasts support a `timeout` option to automatically hide them after a certain amount of time. For accessibility, toasts have a minimum timeout of 5 seconds, and actionable toasts will not auto dismiss. Timers will pause when the user focuses or hovers over a toast.

```tsx render expanded
"use client";
import {Button, UNSTABLE_ToastQueue as ToastQueue} from '@react-spectrum/s2';

<Button
  onPress={() => ToastQueue.positive('Toast is done!', {timeout: 5000})}
  variant="primary">
  Show auto-dismissing toast
</Button>
```

## Programmatic dismissal

Toasts may be programmatically dismissed if they become irrelevant before the user manually closes them. Each method of `ToastQueue` returns a function which may be used to close a toast.

```tsx render expanded
"use client";
import {Button, UNSTABLE_ToastQueue as ToastQueue} from '@react-spectrum/s2';
import {useState} from 'react';

function Example() {
  let [close, setClose] = useState(null);

  return (
    <Button
      onPress={() => {
        if (!close) {
          let close = ToastQueue.negative('Unable to save', {
            onClose: () => setClose(null)
          });
          setClose(() => close);
        } else {
          close();
        }
      }}
      variant="primary">
      {close ? 'Hide' : 'Show'} Toast
    </Button>
  );
}

<Example />
```

### ToastQueue

The `ToastQueue` provides static methods for displaying different types of toasts:

#### Methods

- **`ToastQueue.neutral(children, options?)`** - Shows a neutral toast
- **`ToastQueue.positive(children, options?)`** - Shows a positive/success toast  
- **`ToastQueue.negative(children, options?)`** - Shows a negative/error toast
- **`ToastQueue.info(children, options?)`** - Shows an informational toast

All methods return a function that can be called to programmatically close the toast.

#### Toast Options

- **`actionLabel?: string`** - A label for the action button within the toast
- **`onAction?: () => void`** - Handler that is called when the action button is pressed
- **`shouldCloseOnAction?: boolean`** - Whether the toast should automatically close when an action is performed
- **`timeout?: number`** - Time in milliseconds after which the toast should auto-dismiss (minimum 5000ms)
- **`onClose?: () => void`** - Handler that is called when the toast is closed
